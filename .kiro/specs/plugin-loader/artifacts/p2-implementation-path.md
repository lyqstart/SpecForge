### 2.2 安全边界

| 边界类型 | 防护机制 | 实现方式 |
|----------|----------|----------|
| **进程隔离** | 插件运行在独立子进程 | ProcessManager + spawn |
| **文件系统隔离** | 白名单机制，只允许访问指定目录 | FSWhitelist 验证 |
| **网络隔离** | 网络白名单，限制可访问的域名/端口 | NetworkWhitelist 验证 |
| **资源限制** | 内存、CPU、时间、文件描述符限制 | ResourceMonitor + 终止机制 |
| **环境变量隔离** | 白名单机制，只传递指定的 env | envWhitelist 过滤 |
| **进程派生限制** | 默认禁止 fork，子进程数限制 | maxChildProcesses = 0 |

### 2.3 信任模型

```
信任链（从高到低）：
┌─────────────────────────────────────────┐
│  Daemon 主进程 (完全信任)               │
│       ↓                                 │
│  P0 静态检查 + 权限验证                 │
│       ↓                                 │
│  Sandbox (隔离环境，默认不信任)         │
│       ↓                                 │
│  插件代码 (受限执行)                    │
└─────────────────────────────────────────┘
```

**关键原则**：
1. **默认拒绝**：除非明确授权，否则禁止所有操作
2. **最小权限**：插件只能访问其 `requires` 中声明且被授权的权限
3. **分层防御**：静态检查（P0）+ 运行时隔离（P2）

## 五、风险与缓解

### 5.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| IPC 性能开销 | 高频调用时性能下降 | 批量消息、异步非阻塞 |
| 跨平台差异 | Windows/macOS/Linux 行为不一致 | 平台检测、条件分支 |
| 资源监控精度 | 难以精确获取子进程资源 | 使用 OS 特定 API |
| 进程终止可靠性 | 强制终止可能失败 | 多层终止（SIGTERM → SIGKILL） |

### 5.2 安全风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 白名单配置错误 | 允许非法访问 | 默认最小权限，严格验证 |
| 资源限制绕过 | 插件突破限制 | 多层防御，定期审计 |
| 通信协议漏洞 | 主机被攻击 | 消息验证，限速 |
