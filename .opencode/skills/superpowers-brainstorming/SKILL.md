# Superpowers Brainstorming

## 指令

在开始撰写需求之前，你**必须**从以下 7 个维度逐一进行头脑风暴，每个维度至少列出一个考虑点：

### 1. 业务需求
- 核心业务价值是什么？
- 目标用户场景有哪些？
- 涉及哪些业务流程？
- 与现有业务的关系如何？

### 2. 技术约束
- 平台有哪些限制？
- 依赖关系是什么？
- 兼容性要求有哪些？
- 技术栈的选择约束？

### 3. 用户体验
- 交互方式是否直观？
- 反馈机制是否及时？
- 错误提示是否友好？
- 学习成本是否可接受？

### 4. 安全合规
- 权限控制如何设计？
- 数据保护措施有哪些？
- 审计要求是什么？
- 是否涉及敏感信息处理？

### 5. 运维部署
- 部署方式是什么？
- 监控需求有哪些？
- 故障恢复策略是什么？
- 日志和可观测性如何保障？

### 6. 成本预算
- 资源消耗预估如何？
- API 调用成本是否可控？
- 存储成本如何？
- 是否有性能与成本的权衡？

### 7. 扩展性
- 未来扩展方向是什么？
- 接口是否预留了扩展点？
- 版本兼容性如何保障？
- 是否支持渐进式增强？

## 规则

1. **必须完成所有 7 个维度的头脑风暴后，才能开始撰写结构化需求。**
2. 每个维度至少列出一个具体的考虑点，不得留空。
3. 如果某个维度确实不适用，必须明确说明原因（例如："成本预算：本功能不涉及外部 API 调用，无额外成本"）。
4. 头脑风暴的结果应作为需求分析的输入，确保最终需求文档覆盖了各维度的关键考虑。
