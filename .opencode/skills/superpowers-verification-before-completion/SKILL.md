# Superpowers Verification Before Completion

## 指令

在将任何任务标记为 completed 之前，你**必须**提供以下四类验证证据。缺少任何一类证据，均不得声明任务完成。

### 1. 测试执行结果

- 运行所有与该任务相关的测试（单元测试、属性测试、集成测试）
- 展示完整的测试执行输出
- 确认所有测试通过，无失败用例
- 如有跳过的测试，说明跳过原因

### 2. 构建成功证据

- 确认项目可以成功构建（无编译错误、无类型错误）
- 展示构建命令的执行输出
- 确认无新增的 lint 警告或错误
- 如涉及新依赖，确认依赖安装成功

### 3. 验收标准逐项确认

- 对照 tasks.md 中该任务的验收标准，逐项确认是否满足
- 每个验收标准必须标注：✅ 已满足 或 ❌ 未满足
- 未满足的标准必须说明原因和后续计划
- 所有标准均满足后才能声明完成

### 4. 端到端功能测试结果

- 启动应用并验证核心用户流程可正常操作
- 如果是 Web 应用：启动服务器，验证核心页面可访问，核心操作可执行
- 如果是 CLI 工具：执行核心命令，验证输出正确
- 如果是库：编写集成示例，验证核心 API 可正常调用

## 规则

1. **禁止在没有验证证据的情况下将任务标记为 completed。**
2. **禁止伪造或编造验证证据。** 所有证据必须来自实际执行。
3. 如果无法提供上述任何一类证据（例如：测试环境不可用），必须向 Orchestrator 报告缺失的验证项，由 Orchestrator 决定是否豁免。
4. 验证证据必须是当前代码状态的最新结果，不得使用过期的历史结果。
5. 如果验证过程中发现问题，必须先修复问题再重新验证，不得带着已知问题声明完成。
