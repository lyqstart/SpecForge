---

## 适用场景

```yaml
# SQLite 适合：
#   - 单机应用（桌面软件、CLI 工具、嵌入式设备）
#   - 开发/测试环境（替代真实数据库）
#   - 小型 Web 应用（< 100 并发写入）
#   - 数据文件（替代 CSV/JSON 存储）

# SQLite 不适合：
#   - 高并发写入（SQLite 写操作是串行的）
#   - 多进程共享（文件锁限制）
#   - 需要水平扩展的场景
```

---

## 文件路径规则（覆盖 R1）

**规则**：
1. SQLite 数据库文件路径必须从配置读取，不得硬编码
2. 数据库文件不得提交到 git（加入 `.gitignore`）
3. 测试用数据库文件使用内存模式（`:memory:`）或临时目录

---

## 并发规则

**规则**：
1. 启用 WAL 模式（`PRAGMA journal_mode=WAL`）——支持并发读
2. 设置合理的 busy_timeout（`PRAGMA busy_timeout=5000`）
3. 不得在多线程中共享同一个连接对象
